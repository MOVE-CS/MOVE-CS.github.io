A Comparative Approach to Resurrecting the Market of MOD Vehicular Crowdsensing

1. About this study

With the popularity of Mobility-on-Demand (MOD) vehicles, a new market called MOD-Vehicular-Crowdsensing (MOVE-CS) was introduced for drivers to earn more by collecting road data. Unfortunately, MOVE-CS failed after two years of operation. To identify the root cause, we survey 581 drivers and reveal its simple operation model based on blindly competitive rewards. This model brings most drivers few yields, resulting in their withdrawals. In contrast, a similar market termed MOD-Human-Crowdsensing (MOMAN-CS) remains successful thanks to a complex model based on exclusively customized rewards. Hence, we wonder whether MOVE-CS can be resurrected by learning from MOMAN-CS.
      Despite considerable similarity, we can hardly apply the operation model of MOMAN-CS to MOVE-CS, since drivers are also concerned with passenger missions that dominate their earnings. To this end, we analyze a large-scale dataset of 12,493 MOD vehicles, finding that drivers have explicit preference for short-term, immediate gains as well as implicit rationality in pursuit of long-term, stable profits. Therefore, we design a novel operation model for MOVE-CS, at the heart of which lies a spatial-temporal differentiation-aware task recommendation scheme empowered by submodular optimization. Applied to the dataset, our design would essentially benefit both the drivers and platform, thus possessing the potential to resurrect MOVE-CS.

2. Dataset

We release our dataset on Google Driver, since the data size is larger than 10GB. The dataset consists of two folders, i.e., Vehicle_data and MTurk_data. Please cite this study(IEEE INFOCOM 2022 Submission #1570745558, title "A Comparative Approach to Resurrecting the Market of MOD Vehicular Crowdsensing") when using the data.

  • Vehicle_data: The Vehicle_data folder contains 12,493 txt files, and each file contains a MOD driver's trajectory in a month(3/1/2020-3/31/2020). Each line denotes a data point at the sampling frequency about 15 seconds. The descriptions of the fields of Vehicle_data are as follows:
    •   Field of Dataset     Detailed Information
            Column 1 latitude of MOD vehicles in a time slot
            Column 2 longitude of MOD vehicles in a time slot
            Column 3 speed of MOD vehicles in a time slot
            Column 4 sampling time slot
            Column 5 occupancy status in a time slot
            Column 6 distance from the latest time slot
  • MTurk_data: The MTurk_data folder contains the survey results from 581 participants on Amazon Mechanical Turk, which can be checked on Google Docs.

3. Code

We provide the code of this study on github. As shown in Fig.1, the system of this study consists of 3 modules as follows:

  • Pick-up profit heatmap construction: Pre_Process.py, Attention_Based_RNN.py.
  • Differentiation-aware sensing reward design: Task_Dist.m, Candi_Init.m, Calc_Dis_Time.m, Calc_Profit_Origin.m, Calc_Reward.m.
  • Submodularity-based task recommendation: LSTRec.m.
    •   File Name     Detailed Information
      Pre_process.py pick-up profit historical heatmap construction
      Prediction.py pick-up profit heatmap prediction
      Initialize_task.m sensing task initialization
      Initialize_candidate.m candidate MOD driver initialization
      Calculate_consumption.m calculation of each vehicle-task pair's consumptin
      Calculate_original_profit.m calculating original profits without LSTRec
      Calculate_reward.m calculation of sensing rewards for MOD drivers
      LSTRec.m task recommendation algorithm of LSTRec
      Emulation.m emulation code of LSTRec system
